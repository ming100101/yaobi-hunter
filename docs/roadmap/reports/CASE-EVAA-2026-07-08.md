# Live case study — EVAA +136% (2026-07-07 pump) — 「更早」問題嘅實例

**日期**: 2026-07-08(pump 發生 2026-07-07)· **性質**: live retrospective,N=1,建設性但**未 backtest**(照 IRON RULE 唔 ship)
**數據**: Binance fapi 公開 endpoint(1H/5m klines + openInterestHist),`scratchpad/evaa*.mjs`

## TL;DR

EVAA 由 $0.84 拉到 $2.98(+210% low→peak)。**唯一可誠實偵測嘅一刻係 07-07 09:00 嘅 +22.6% 巨量 candle(vol $2M→$19M,10×)** —— 嗰陣仲有 **+84% 喺前面**,所以「fire 就已經太遲」對 EVAA **唔成立**,只要你 fire 喺 raw 量能點火。**真正遲嘅係所有 keyed 喺點火下游嘅嘢**:OI-confirmation(增/擴)同 ⚡ flush-reclaim geometry —— 佢哋結構上一定喺 raw candle 之後先 fire。呢個**收窄**咗通宵結論:coincident 訊號之中,**raw 價+量點火最早;每加一個條件都推遲 fire。**

## 對用戶「每次都太遲」嘅解讀

1. **Fire 喺 raw 點火(09:00)= 仲有 +84%,唔遲。** 用戶感覺「太遲」大機會係因為佢實際 fire 點喺**下游**:
   - **⚡ flush-breakout** 要「先 flush 再 reclaim」嘅 geometry,喺 markup 中段先成形(通宵已證 flush 係**負** filter,matched ×0.5)。
   - **增/擴** 要 OI 擴張確認,但 OI 喺 09:00 仲 flat,要等佢追上(+356% 係 markup 期間先發生)→ OI-based detector 結構上遲過 raw candle。
   - 記憶中 EPIC case:增會喺點火鐘(10:00)fire —— 即係增其實近點火,唔算太遲;最遲嗰個係 ⚡。
2. **靜默 grind($0.84→$1.19,+42%,33h)偵測唔到,唔係 detector 差。** 嗰段冇量、細 candle,同垃圾 drift 一模一樣。想 catch 嗰段 = catch 所有 drift = 純噪音。呢個就係「冇 free lunch」:唯一 distinguishing 訊號(量能爆發)喺 09:00 先出現,唔會更早。

## 建設性假設(**未 backtest,唔 ship**)

**為咗「更早」:減條件,唔好加。** 最 raw 嘅「hourly close +≥15% 且 volume ≥5× 近期均值」點火 candle 係最早可 fire 點,早過 ⚡ flush 同 OI-confirmation。呢個同通宵結論一致(⚡ flush 係負 filter、特殊條件唔 beat raw geometry、增 fire 喺 breakout)。

**下一步(gate 過先算):**
1. Harness 加一個「pure volume-ignition」mode(+≥X%/1h ∧ vol z≥Y,冇 flush / 冇 OI gate),對歷史 pump 集量度 captures%(earliness)vs ⚡/增/擴。假設:captures 高過 ⚡(40%)、增/擴(30%)。
2. **同時量 false-positive 成本**:純量能點火會喺好多失敗 pump 都 fire —— 呢個正正係點解要 state-matched backtest,唔可以睇 EVAA 一個 case 就 ship。lift ≥×1.3 + robustness 先講。
3. 如果過 gate → 呢個就係「比 ⚡/增/擴 更早」嘅實質答案(fire 喺 raw 點火,唔等下游確認)。如果唔過 → 確認咗「raw 點火 = 純噪音,下游條件係買 edge 嘅代價」,亦係有用嘅定案。

## Caveat(誠實)

N=1。EVAA 係極端 case(+210%、from-low 垂直、無 grind 量能)。單一 case **唔能夠**建立 detector —— 佢只係生成假設。真正結論要 backtest 全 pump 集嘅 captures% + false-positive。**照 IRON RULE:今次零 code / 零 ship,純 case study + 假設,俾用戶醒起身睇。**
